## Описание<a id="description"></a>

В проекте реализовано веб-приложение на Django, которое позволяет пользователям управлять рассылками сообщений для
клиентов. Приложение включает в себя функциональность для создания, просмотра, редактирования и удаления рассылок, а
также отправки сообщений по требованию.

## Приложение Mailings<a id="mailings_app"></a>

Приложение создано для создания/редактирования/удаления получателей рассылок, сообщений и рассылок. Также реализована
возможность отправки рассылок по требованию. Реализована страница с отчетом по кажой рассылке и главная страница со
статистикой.

Ниже будут описаны модели, контроллеры + ссылки, админ-панель, шаблоны и пользовательские команды.

### Модели<a id="mailings_models"></a>

В приложении создано 4 модели:
- Recipient - получатель рассылки. Содержит поля email, full_name, comment, owner(кем создан).
- Message - сообщение. Содержит поля subject(тема письма), body(тело письма), owner(кем создан).
- Mailing - рассылка. Содержит поля created_at(дата отправки), finished_at(дата окончания отправки), status, message, 
recipient owner(кем создан). При создании рассылка получает статус 'Создана', при отправке 'Запущена', при отправке в
архив 'Завершена'. После отправки рассылки в архив вернуть обратно рассылку может только админ.
- Attempt - результат отправки рассылки на почту. Содержит поля created_at(дата отправки), status,
server_response(ответ сервера), recipient. Содержит 2 статуса: успешно отправлена и не успешно(ошибка).

### Контроллеры и ссылки<a id="mailings_controllers"></a>

1. Контроллер MainTemplateView для отображения главной страницы сайта. Для пользователей с ролью 'Админ' и
'Manager(менеджер)' отображается статистика по всем записям. Для пользователей с ролью 'Пользователь' только свои
созданные. Реализовано с помощью добавления контекстных переменных.

```

2. Контроллеры Recipient.

получателя.

редактирование получателя.
   - Контроллер RecipientDeleteView для удаления получателя. Роль 'Manager(менеджер)' - запрещено удалять получателя.
   - Контроллер RecipientUpdateView для обновления информации о получателе. Роль 'Manager(менеджер)' - запрещено
обновлять информацию о получателе.

```

Ссылка для контроллера CreateView: адрес/mailings/recipient/create/
Ссылка для контроллера DetailView: адрес/mailings/recipient/<int:pk>/detail/
Ссылка для контроллера DeleteView: адрес/mailings/recipient/<int:pk>/delete/

   - Контроллер MessageUpdateView для обновления данных сообщения.

```

Ссылка для контроллера DetailView: адрес/mailings/message/<int:pk>/detail/
Ссылка для контроллера DeleteView: адрес/mailings/message/<int:pk>/delete/
Ссылка для контроллера UpdateView: адрес/mailings/message/<int:pk>/update/

### Админ-панель<a id="mailings_admin"></a>

Реализована админ-панель для 4 моделей:
1. RecipientAdmin панель. Отображаются поля: "id", "email", "full_name". Фильтрация по полю "full_name". Поиск по полю:
"email", "full_name".
2. MessageAdmin панель. Отображаются поля: "id", "subject", "body". Фильтрация по полю "subject". Поиск по полю
"subject".
3. MailingAdmin панель. Отображаются поля: "id", "created_at", "status", "message". Фильтрация по полю:
"created_at", "status". Поиск по полю "message".
4. AttemptAdmin панель. Отображаются поля: "id", "created_at", "status", "server_response". Фильтрация по полю:
"created_at", "status". Поиск по полю "server_response".

## Приложение Users<a id="users_app"></a>

Приложение создано для регистрации/авторизации пользователя на сайте(реализована возможность восстановления пароля).
Также реализована страница с профилем пользователя и возможность блокировки/разблокировки пользователя.

Ниже будут описаны модели, контроллеры + ссылки, админ-панель, шаблоны и пользовательские команды.

### Модели<a id="users_models"></a>

В приложении создана модель User - пользователь. Содержит поля email, avatar, phone, country, token.

### Контроллеры и ссылки<a id="users_controllers"></a>

1. Контроллер RegisterView и email_verification для регистрации пользователя. При регистрации на почту отправляется
письмо с токеном. Перейдя по ссылке пользователь подтверждает свою почту и успешно проходит регистрацию. Всем
зарегистрированным пользователям присваивается группа прав 'User(пользователь)'.

```
Ссылка для контроллера: адрес/users/register/
Ссылка для подтверждения почты через токен: адрес/users/email-confirm/<str:token>/
Ссылка для авторизации: адрес/users/login/
Ссылка для выхода из аккаунта: адрес/users/logout/
```

2. Контроллеры User.
   - Контроллер UsersListView для отображения страницы со всеми пользователями сервиса. Используется queryset:
если 'Админ' - то видит всех менеджеров и пользователей, если 'Manager(менеджер)' - всех пользователей.
   - Контроллер UserDetailView для просмотра информации пользователя.
   - Контроллер UserUpdateView для обновления информации о пользователи.

```
Ссылка для контроллера ListView: адрес/users/users/
Ссылка для контроллера DetailView: адрес/users/user/<int:pk>/detail/
Ссылка для контроллера UpdateView: адрес/users/user/<int:pk>/update/
```

3. Контроллеры BlockUserView и UnlockUserView.
   - Контроллер BlockUserView для блокировки пользователя на сервере. Доступно для пользователей с правами доступа
'Manager(менеджер)' и 'Админ'. При этом 'Manager(менеджер)' может блокировать только пользователей.
   - Контроллер UnlockUserView для разблокировки пользователя на сервере. Доступно для пользователей с правами доступа
'Manager(менеджер)' и 'Админ'. При этом 'Manager(менеджер)' может разблокировать только пользователей.

```
Ссылка для контроллера BlockUserView: адрес/users/user/<int:pk>/block/
Ссылка для контроллера UnlockUserView: адрес/users/user/<int:pk>/unblock/

## Группы прав<a id="groups_rights"></a>

В приложении используется 3 группы прав:
1. Группа прав Admin(Админ) - может смотреть/создавать/удалять/редактировать все данные на сайте. Может блокировать и
разблокировать всех менеджеров и пользователей сайта.
2. Группа прав Manager(Менеджер). **Важно - при добавлении данной группы для корректной работы приложения необходимо
снять с пользователя группу прав (User).** Может просматривать всех получателей и рассылки. Может добавлять рассылки
в архив, блокировать и разблокировать пользователей сайта.
3. Группа прав User(Пользователь) - может смотреть/создавать/удалять/редактировать своих получателей, сообщения,
рассылки, личный профиль. **Данная группа прав автоматически присваивается при регистрации.**

## Запуск и тестирование проекта<a id="launch"></a>

1. После установки и настройки проекта в консоль введите python/python3 manage.py runserver для запуска сервера
2. Откройте сервер (ссылка в консоли: http://127.0.0.1:8000)
3. Для просмотра главной страницы сайта используйте http://127.0.0.1:8000/mailings/main/
4. Для входа в админ-панель используйте http://127.0.0.1:8000/admin/
